Featured Project
FDA Complete Response Letter — Public Safety Risk Analysis

The FDA publishes Complete Response Letters — formal documents issued when a drug application has critical deficiencies. They're public, detailed, and largely unread at scale. Using an automated ETL pipeline, an AI-assisted scoring methodology based on an internally created scoring rubric, and an interactive dashboard, it turns hundreds of unstructured regulatory letters into a quantified public safety risk index — by company, by facility, over time.

It's a full-stack data project: cloud infrastructure, a production pipeline, AI integration, custom metric design, and interactive visualization — built end to end.

Metric Background
What is a Complete Response Letter?

A Complete Response Letter is the FDA's formal response when a drug or biologic application cannot be approved in its current form, meaning the deficiencies identified put public safety at risk. It describes every deficiency the agency identified in detail. Many are published publicly by the FDA, but they're long, unstructured text, and published at a volume that makes manual review impractical.

This project ingests every available CRL from the FDA CRL Transparency API, uses AI to extract structured risk signals, calculates scores from the AI's output, and visualizes the results. Additionally, the dashboard also surfaces the AI's reasoning behind each scoring decision, traceable back to the source text.

How the score is calculated

Each letter is scored across four dimensions:

Deficiency Severity

How serious were the cited manufacturing, clinical, or safety deficiencies? Scored 0 to −50, weighted by drug type.

Drug Type Multiplier

A multiplier reflecting the risk profile of the drug category. A critical care drug with deficiencies carries more public safety weight than a cosmetic product.

Facility Inspection

Were facility-level inspection failures cited? Contributes up to −25 points.

Outcome Severity

Did the letter reference adverse patient outcomes or safety failures in trials? Contributes up to −25 points.

Score = 100 + deficiency_score + facility_score + outcome_score
Higher score = lower risk. Scores below 50 indicate elevated concern.

Roll-up logic

Where a company has received multiple CRLs at the same facility, scores are averaged across letters. Each additional letter beyond the first applies a 10-point penalty, reflecting the elevated risk associated with repeat regulatory action. The roll-up score is floor-capped at 0.

Limitations

The CRL dataset is a growing but incomplete archive — not all letters issued by the FDA are currently published. Redactions in source documents may limit the AI's ability to assess the full scope of cited deficiencies. Scores reflect an assessed risk signal based on regulatory language, not an official FDA determination.

ETL Pipeline

This project runs on a fully automated, cloud-native pipeline that checks the FDA's CRL database for new letters and incrementally updates risk scores on a monthly basis.

Each run

Pulls new letters from the FDA Transparency API
Checks against existing records to identify only net-new data
Geocodes new facility addresses via Google Maps
Sends letters in batches of 10 to Claude Haiku for AI scoring
Maps categorical scores to numeric values via a reference table
Rebuilds the company-level roll-up
