Featured Project
FDA Public Safety Risk Analysis

Each dot on the map is a pharmaceutical company that received an FDA Complete Response Letter — a formal notice that a drug application has safety or quality problems. The color shows how risky that company's track record is. Red = high risk. Green = low risk. Click any dot to see the details.

How to read this: Each point is scored 0–100. Higher = safer. Scores factor in how serious the deficiencies were, what type of drug was involved, whether facility inspections failed, and whether the company has repeat offenses. Hover for company details.
01 — The Pipeline

Automated data collection and processing

Every month, an AWS scheduled task pulls new FDA letters, checks them against existing records, geocodes facility addresses, and feeds everything into Databricks. No manual steps — it runs unattended and recovers from failures automatically.

Read the full technical documentation →
02 — The AI Scoring Engine

LLM-powered risk evaluation

Each letter is sent to Claude, which reads the full text and scores it against a custom four-category rubric. The AI classifies deficiency severity, drug risk type, facility inspection results, and patient outcome impact — then its reasoning is stored alongside the scores for traceability.

See how scores are calculated →
03 — The Stack

Cloud-native, fully containerized

Python application containerized with Docker, deployed on AWS ECS Fargate, triggered monthly by EventBridge. Secrets managed through AWS Secrets Manager. Data stored in Databricks Delta Lake. All source code on GitHub.

Scoring Methodology
How the risk score works

What's a Complete Response Letter?

When the FDA reviews a drug application and finds problems — safety concerns, manufacturing issues, incomplete data — they send a Complete Response Letter explaining what needs to be fixed before the drug can be approved. These letters are public, detailed, and published at a volume that makes manual review impractical.

This project uses AI to read each letter, extract the key risk signals, and produce a numeric score.

Four scoring dimensions

Deficiency Severity

How serious were the cited problems? Manufacturing failures, missing clinical data, and safety gaps are weighted more heavily.

Drug Type Multiplier

A critical-care injectable with problems carries more public risk than a cosmetic product. This multiplier reflects that.

Facility Inspection

Were there facility-level inspection failures? This captures manufacturing environment risk.

Outcome Severity

Did the letter reference adverse patient outcomes or safety failures in clinical trials?

Score = 100 + deficiency + facility + outcome
Higher score = lower risk. Each dimension deducts from 100 based on severity.

Score interpretation

ScoreRisk Level
91 – 100Minimal
71 – 90Low
51 – 70Moderate
36 – 50Elevated
16 – 35High
0 – 15Critical

Company roll-up

When a company has multiple letters at the same facility, scores are averaged. Each additional letter beyond the first applies a 10-point penalty — repeat regulatory action signals higher risk. Scores are floored at 0.

Limitations

The FDA's published CRL archive is growing but incomplete. Redactions in source documents may limit what the AI can assess. Scores reflect an estimated risk signal based on regulatory language — not an official FDA determination.

Documentation
ETL Pipeline

The pipeline runs fully automated on AWS. Once a month, it checks for new FDA letters and incrementally updates all risk scores.

What happens each run

Pulls new letters from the FDA Transparency API
Checks against existing records to identify only new data
Geocodes new facility addresses via Google Maps
Sends letters in batches to Claude for AI scoring
Maps AI output to numeric scores via reference tables
Rebuilds the company-level roll-up with repeat-offense penalties
